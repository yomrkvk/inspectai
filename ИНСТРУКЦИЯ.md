# ЧАСТЬ 1. ЧТО УСТАНОВИТЬ ОДИН РАЗ

Эти программы нужно поставить **один раз**. После этого — просто открывать терминал и запускать.

---

## 1.1 Docker Desktop

**Для чего:** запускает весь проект (базу данных, сервер, сайт) одной командой.

**Windows:**
1. Перейдите на https://www.docker.com/products/docker-desktop/
2. Нажмите **"Download for Windows"**
3. Запустите скачанный `.exe`, нажимайте **Next → Install → Finish**
4. Перезагрузите компьютер
5. После перезагрузки в трее (правый нижний угол) появится иконка Docker — кит 🐳
6. Подождите пока иконка перестанет крутиться — Docker готов

**Mac:**
1. Перейдите на https://www.docker.com/products/docker-desktop/
2. Выберите версию под ваш чип — **Apple Silicon** (M1/M2/M3) или **Intel**
3. Установите `.dmg` файл как обычное приложение
4. Запустите Docker Desktop из Finder → Applications
5. Дождитесь надписи **"Docker Desktop is running"**

**Проверка что всё работает (в терминале):**
```
docker --version
```
Должно ответить что-то вроде: `Docker version 27.x.x`

---

## 1.2 Git

**Для чего:** отслеживает изменения в коде, загружает на GitHub.

**Windows:**
1. Перейдите на https://git-scm.com/download/win
2. Скачается автоматически — запустите установщик
3. На всех экранах нажимайте **Next** (настройки по умолчанию подходят)
4. На экране с редактором выберите любой — можно оставить Vim или выбрать Notepad
5. Нажмите **Install → Finish**

**Mac:**
```
xcode-select --install
```
Появится диалог — нажмите **Install**. Займёт 5–10 минут.

**Проверка:**
```
git --version
```
Ответит: `git version 2.x.x`

---

## 1.3 Как открыть терминал

**Windows:**
- Способ 1: нажмите `Win + R`, введите `cmd`, нажмите Enter
- Способ 2: в папке с проектом — правая кнопка мыши на пустом месте → **"Открыть в терминале"** или **"Git Bash Here"**
- Способ 3: нажмите `Win + X` → **"Терминал"**

**Mac:**
- Нажмите `Command + Пробел`, введите `Terminal`, Enter
- Или: Finder → Applications → Utilities → Terminal

# ЧАСТЬ 2. ПЕРВЫЙ ЗАПУСК ПРОЕКТА

---

## 2.1 Распакуйте архив

Скачайте `inspectai_mobile.zip` и распакуйте:
- **Windows:** правая кнопка на файле → **"Извлечь всё"** → выберите папку (например `C:\Projects\`) → **"Извлечь"**
- **Mac:** двойной клик по архиву — распакуется рядом

Получится папка `inspectai`. Запомните где она.

---

## 2.2 Откройте терминал В папке проекта

**Windows:**
1. Откройте папку `inspectai` в Проводнике
2. Правая кнопка на **пустом месте** (не на файле) → **"Открыть в терминале"**
3. Если такого нет — нажмите на адресную строку вверху, введите `cmd`, Enter

**Mac:**
1. Откройте Terminal
2. Введите `cd ` (с пробелом после cd), затем перетащите папку `inspectai` из Finder прямо в Terminal
3. Нажмите Enter

**Проверка — введите:**
```
ls
```
Должны увидеть: `backend  docker-compose.yml  frontend  ml_pipeline  scripts`

---

## 2.3 Создайте файл настроек

Введите в терминале:

**Windows (cmd):**
```
copy backend\.env.docker .env
```

**Mac / Windows (Git Bash):**
```
cp backend/.env.docker .env
```

Теперь откройте файл `.env` в папке `inspectai` — любым текстовым редактором (Блокнот, VS Code, TextEdit).

Найдите строку:
```
SECRET_KEY=замените-на-случайный-ключ-минимум-32-символа
```

Замените на любой набор букв и цифр длиной больше 32 символов. Например:
```
SECRET_KEY=inspectai-my-project-2026-dudareva-polina-key
```

Сохраните файл.

> Всё остальное (ключ Яндекс.Карт, пароль базы данных) уже заполнено — не трогайте.

---

## 2.4 Запустите проект

В терминале введите:
```
docker compose up --build
```

**Что будет происходить:**
- Docker начнёт скачивать образы (Python, PostgreSQL, Node.js) — это займёт **5–20 минут** в первый раз
- Вы увидите много текста — это нормально
- Ждите строку: `Uvicorn running on http://0.0.0.0:8000` — значит всё запустилось

**Не закрывайте этот терминал** — он должен работать пока вы используете сайт.

---

## 2.5 Откройте сайт

Откройте браузер и перейдите на:
```
http://localhost:5173
```

Войдите с данными:
- Логин: `admin`
- Пароль: `admin123`

---

## 2.6 Остановка и повторный запуск

**Остановить:** нажмите `Ctrl + C` в терминале с запущенным Docker

**Запустить снова** (быстро, без --build):
```
docker compose up
```

**Запустить в фоне** (терминал освободится):
```
docker compose up -d
```

**Остановить фоновый:**
```
docker compose down
```

# ЧАСТЬ 3. ОТКРЫТЬ САЙТ НА ТЕЛЕФОНЕ

Два способа — выбирайте любой.

---

## Способ А: Через локальную сеть (быстрее, без интернета)

Работает если телефон и компьютер подключены к **одной Wi-Fi сети**.

**Шаг 1.** Узнайте IP-адрес компьютера:

**Windows:**
1. Нажмите `Win + R`, введите `cmd`, Enter
2. Введите: `ipconfig`
3. Найдите раздел вашего Wi-Fi адаптера
4. Строка **"IPv4 Address"** — это ваш IP, например `192.168.1.100`

**Mac:**
1. Откройте Terminal
2. Введите: `ifconfig | grep "inet " | grep -v 127`
3. Найдите строку вида `inet 192.168.1.100` — это ваш IP

**Шаг 2.** Убедитесь что проект запущен (`docker compose up`)

**Шаг 3.** На телефоне откройте браузер и введите:
```
http://192.168.1.100:5173
```
(замените 192.168.1.100 на ваш реальный IP)

Готово — сайт работает на телефоне.

---

## Способ Б: Через ngrok (работает из любой сети, нужен интернет)

Ngrok создаёт публичную ссылку вида `https://abc123.ngrok-free.app` — её можно отправить кому угодно.

### Установка ngrok (один раз):

**Windows:**
1. Перейдите на https://ngrok.com/download
2. Выберите **Windows**, скачайте `.zip`
3. Распакуйте — получите файл `ngrok.exe`
4. Переместите `ngrok.exe` в папку `C:\Windows\System32\` (чтобы работал из любой папки)
   - Или просто запускайте из папки где лежит

**Mac:**
```
brew install ngrok/ngrok/ngrok
```
Если нет Homebrew — установите с https://brew.sh/ или скачайте вручную с ngrok.com

### Регистрация на ngrok (один раз, бесплатно):

1. Зайдите на https://ngrok.com
2. Нажмите **"Sign Up"** — зарегистрируйтесь (можно через Google)
3. После входа перейдите в раздел **"Your Authtoken"** (или Dashboard → Getting Started → Your Authtoken)
4. Скопируйте токен — выглядит как: `2abc123def456_ghijklmnop789`

### Настройка токена (один раз):

```
ngrok config add-authtoken ВАШ_ТОКЕН
```

### Использование:

1. Убедитесь что проект запущен (`docker compose up`)
2. **Откройте новый терминал** (не закрывая первый)
3. Введите:
```
ngrok http 5173
```

Увидите:
```
Forwarding    https://abc123def456.ngrok-free.app -> http://localhost:5173
```

Ссылку `https://abc123def456.ngrok-free.app` можно:
- Открыть на телефоне
- Отправить однокурснице/преподавателю
- Показать во время защиты прямо с телефона

**Ссылка работает пока работает ngrok.** При каждом новом запуске адрес меняется.

### Добавить сайт на главный экран телефона:

**iPhone (Safari):**
1. Откройте ссылку в Safari (не в Chrome)
2. Нажмите кнопку **Поделиться** (квадрат со стрелкой вверх)
3. Прокрутите вниз → **"На экран «Домой»"**
4. Нажмите **"Добавить"**

**Android (Chrome):**
1. Откройте ссылку в Chrome
2. Три точки **⋮** в правом верхнем углу
3. **"Добавить на главный экран"**

Теперь InspectAI открывается как полноэкранное приложение.

# ЧАСТЬ 4. ЗАГРУЗИТЬ НА GITHUB

---

## 4.1 Создайте аккаунт GitHub (если нет)

1. Зайдите на https://github.com
2. Нажмите **"Sign up"**
3. Введите email, пароль, имя пользователя
4. Подтвердите email

---

## 4.2 Создайте репозиторий

1. Войдите в GitHub
2. Нажмите **+** в правом верхнем углу → **"New repository"**
3. Заполните:
   - **Repository name:** `inspectai`
   - **Description:** `Система контроля благоустройства на основе компьютерного зрения`
   - Выберите **Private** (приватный — только вы видите) или **Public**
   - **НЕ** ставьте галочки "Add README", "Add .gitignore", "Add license" — они уже есть в проекте
4. Нажмите **"Create repository"**

GitHub покажет страницу с инструкцией. Скопируйте адрес репозитория — выглядит так:
```
https://github.com/ваш-логин/inspectai.git
```

---

## 4.3 Настройте Git (один раз)

В терминале введите (замените на ваши данные):
```
git config --global user.name "Дударева Полина"
git config --global user.email "ваш@email.com"
```

---

## 4.4 Создайте Personal Access Token (вместо пароля)

GitHub больше не принимает обычные пароли при загрузке кода. Нужен токен.

1. GitHub → ваша аватарка (правый верхний угол) → **"Settings"**
2. Прокрутите вниз, слева найдите **"Developer settings"**
3. **"Personal access tokens"** → **"Tokens (classic)"**
4. **"Generate new token"** → **"Generate new token (classic)"**
5. Заполните:
   - **Note:** `inspectai`
   - **Expiration:** выберите `90 days` или `No expiration`
   - **Scopes:** поставьте галочку на **`repo`** (самый первый пункт)
6. Нажмите **"Generate token"**
7. Скопируйте токен — он показывается **только один раз**!
   Сохраните в блокноте или заметках на телефоне.

Токен выглядит так: `ghp_abc123def456ghijklmnopqrstuvwxyz`

---

## 4.5 Загрузите проект на GitHub

Откройте терминал **в папке inspectai** и введите команды одну за другой:

```
git init
```
```
git add .
```
```
git commit -m "Первоначальная загрузка InspectAI"
```
```
git branch -M main
```
```
git remote add origin https://github.com/ВАШ-ЛОГИН/inspectai.git
```
```
git push -u origin main
```

При последней команде Git спросит:
- **Username:** ваш логин на GitHub
- **Password:** вставьте **токен** (не пароль от аккаунта!)

На Mac токен вставляется через `Command + V`, на Windows через правую кнопку мыши или `Ctrl + V` в терминале.

После успеха обновите страницу репозитория на GitHub — там появится весь проект.

---

## 4.6 Сохранить пароль чтобы не вводить каждый раз

```
git config --global credential.helper store
```

После этой команды следующий `git push` попросит логин/токен в последний раз — сохранит навсегда.

# ЧАСТЬ 5. ЕЖЕДНЕВНАЯ РАБОТА С ПРОЕКТОМ

---

## Как запускать каждый день

1. Открыть Docker Desktop — подождать пока запустится
2. Открыть терминал в папке `inspectai`
3. Ввести: `docker compose up`
4. Открыть браузер: `http://localhost:5173`

---

## Полезные команды

| Что делать | Команда |
|-----------|---------|
| Запустить | `docker compose up` |
| Запустить в фоне | `docker compose up -d` |
| Остановить | `docker compose down` |
| Перезапустить после изменений кода | `docker compose up --build` |
| Посмотреть логи | `docker compose logs -f` |
| Логи только сервера | `docker compose logs -f backend` |
| Зайти в базу данных | `docker compose exec db psql -U inspectai -d inspectai` |
| Полный сброс (удалить всё) | `docker compose down -v` |

---

## Если что-то не работает

**Сайт не открывается:**
- Проверьте что Docker Desktop запущен
- Проверьте что в терминале нет ошибок (красного текста)
- Подождите 30–60 секунд после запуска

**"Port already in use":**
```
docker compose down
docker compose up
```

**Полный сброс (если совсем всё сломалось):**
```
docker compose down -v
docker compose up --build
```
Внимание: `-v` удаляет базу данных — все загруженные фото и результаты удалятся.

**Сайт открывается но не входит:**
- Логин: `admin`, пароль: `admin123`
- Попробуйте полный сброс выше

---

## Что НЕ нужно делать

- Не редактируйте файлы внутри `node_modules/` или `__pycache__/`
- Не коммитьте файл `.env` в Git (он в `.gitignore` — защита паролей)
- Не удаляйте папку `.git` — это ваша история изменений
- Не запускайте `docker compose up --build` каждый раз — только когда меняли зависимости
